Every element, every state

A zero-dependency storybook for Wisp's drop-in elements. Live stories talk to this dev server for real — create accounts, sign out, watch the events land below. Frozen stories are posed snapshots of internal states (busy, error, the shown-once panel) so you can see them without triggering them.

<wisp-button>

One element, four actions. Handles its own busy state (aria-busy, disabled) and shows failures inline. Emits wisp-success or wisp-signout.

Live — all four actions

Default labels. Sign-up/sign-in open a real passkey prompt.

<wisp-button action="signup"></wisp-button>
<wisp-button action="signin"></wisp-button>
<wisp-button action="guest"></wisp-button>
<wisp-button action="signout"></wisp-button>

Custom label

The label attribute overrides the default text. Live.

<wisp-button action="guest"
  label="Just let me in"></wisp-button>

Theming

Elements ship unstyled-but-decent and take the host page's brand two ways: CSS custom properties for the quick pass, ::part() for full control. Every stage on this page already maps --wisp-* to the page palette — including dark mode, which the theme toggle above proves.

CSS variables

Accent, text, border, error color, radius.

.mine {
  --wisp-accent: #e11d48;
  --wisp-accent-text: #fff;
  --wisp-radius: 999px;
}

::part() override

Restyle any internal piece: button, input, panel, code, chip, error.

wisp-button::part(button) {
  background: transparent;
  border: 2px solid var(--brand);
  text-transform: uppercase;
}
